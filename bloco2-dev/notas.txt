# QUESTÕES DE SÍNTESE

# 1. O nm mostrou printf como "U" (undefined). Em qual slide da
#    apresentação esse mesmo problema apareceu? O que o ligador
#    faz com esse símbolo?

Resposta 1:  O ligador atua como um "integrador" final. Como o código do printf não está no meu arquivo (ele pertence à biblioteca padrão do sistema), o compilador apenas sinaliza que ele é necessário. O papel do ligador é encontrar onde essa função está guardada no sistema e conectar os pontos, preenchendo o endereço correto para que o programa saiba o que executar quando o printf for chamado.


# 2. Compare soma.i com soma.s: qual das duas saídas é análoga
#    ao "código objeto" do montador histórico? Justifique.

Resposta 2:  Na realidade, o computador não executa nenhum desses três arquivos diretamente. O .i ainda é texto, o .s é uma representação para humanos e o .o é código de máquina, mas está "solto" e incompleto. O processador só consegue rodar o executável final (o arquivo "soma"), que foi gerado pelo ligador com todas as dependências resolvidas e organizado para o sistema operacional.
