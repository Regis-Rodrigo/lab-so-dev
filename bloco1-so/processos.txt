# PARTE B — Processo filho
# Cole a saída dos comandos e responda:

sleep 5m &
  PID do filho: 1300

cat /proc/[FILHO]/status | grep -E "Pid|PPid|State"
  Pid: 1300
  PPid: 1234
  State: S (sleeping)

ps -o pid,ppid,stat,cmd (antes do kill)
    PID  PPID STAT CMD
   1234  1200 S    bash
   1300  1234 S    sleep 5m

ps -o pid,ppid,stat,cmd (depois do kill)
    PID  PPID STAT CMD
   1234  1200 S    bash


# PARTE C — Questões de síntese

# 1. O campo PPID em /proc/$$/status indica o processo pai do shell.
#    Execute ps -o pid,ppid,cmd e localize o par shell -> processo pai.
#    O que esse encadeamento sugere sobre como os processos sao
#    criados no Linux?

Resposta 1:
O encadeamento mostra que os processos no Linux são criados a partir de outros processos já existentes. Cada processo possui um processo pai, formando uma hierarquia em árvore.


# 2. O arquivo /proc/<PID>/status mostra o campo State com uma letra
#    (S, R, Z...). Depois de executar kill $FILHO e rodar ps novamente,
#    o que aconteceu com o processo filho? O que isso te faz pensar  
#    sobre a vida de um processo?

Resposta 2:
Depois do comando kill, o processo filho deixou de aparecer no ps, indicando que foi encerrado. Isso mostra que processos possuem um ciclo de vida: são criados, executam tarefas e depois terminam.
